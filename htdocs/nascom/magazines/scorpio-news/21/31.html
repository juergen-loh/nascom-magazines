C0:BAK> MCOPY MDSK:=BAK:FILE1.TYP,FILE2.TYP
works O.K.

M0:MDSK> MCOPY=B3:FIL1.TYP,FIL2.TYP
FIL1 copied O.K. FIL2 gives SRC=DEST err.

C0:BAK> MCOPY MDSK:=B3:FIL1.TYP,FIL2.TYP
FIL1 copied O.K. but MCOPY tries to find FIL2 on C0: and gives a Not Found err.

The problems seem to mainly arise if one is not logged into the source directory when using MCOPY. If anyone has a patch for this I would be interested to hear from them, since it can cause batch processing to abort.
